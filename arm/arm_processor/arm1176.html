  1. ARMv6 (legacy) vs ARMv7-A

    There is another version of ARMv6-M, which is used for current ARM Cortex-M micro-architecture. This part introduces the old ARMv6 that is used by ARM11.

    ARM ALU and RISC

    1. load/store architecture, where data-processing operations only operate on register contents, not directly on memory contents.
    2. auto-increment and auto-decrement addressing modes to optimize program loops
    3. simple addressing modes, with all load/store addresses being determined from register contents and instruction fields only.

    Difference between ARMv6 and ARMv7-A

    1. ARMv6 use the first generation of Thumb, which is the incomplete, prue 16-bit thumb. (code switches between thumb mode and arm32)
      ARMv7 use the Thumb-2, which includes 16-bit and 32-bit thumb. It is perhaps the reason that why Ubuntu does not support ARMv6.

    Technologies

    1. TrustZone
    2. Jazelle: directly run java bytecode on the processor. Hardware (Circuit) translates bytecode to the actually runnable code.
    3. ThumbEE: a variant of the Thumb instruction set that minimizes the code size overhead of a Just-In-Time (JIT) or Ahead-Of-Time (AOT) compiler
    4. SIMD (Single Instruction Multiple Data): DSP instruction that operate on 16-bit or 8-bit data values in 32-bit registers

  2. BUS, AMBA (advanced microprocessor bus architecture)

    1. AMBA is used to connect different functional blocks, e.g.
    2. AMBA (1996) ASB and APB, AMBA 2 (1999) AHB, AMBA 3(2003) AXI, AMBA 4(2010) AXI4 and ACE, AMBA 5 (2013) coherent hub interface

  3. ARM1176JZF-S

    Features

    1. 8 stage pipeline
    2. MMU
    3. 7 operating mode: User, Supervisor, fast interrupt, normal interrupt, abort, system, undefined, secure monitor
    4. Harvard Architecture
