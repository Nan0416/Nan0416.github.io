  1. Introduction

    The trivial alogrithms for BFS is Queue, DFS is recursion.

    BFS only has one order, DFS has 3 different orders (preorder, inorder and postorder) [Inorder only works with binary tree]

    1. BFS vs. DFS

      BFS 很老实, it iterates the entire graph, tree, matrix layer by layer. Finally, it can traverse every node. A well designed DFS is smart, it can also find a target very quick in O(h)=O(lgn). It can also iterates every node in O(n).

      BFS most likely applied with undirected graph, (directed) binary tree.

    2. BFS vs. Union Find

      When finding connected component in a undirected graph, or matrix. Both union find and BFS can be applied, but union find is better.

    3. Implementation of BFS

      BFS is implemented with a queue, first add the root to queue.

      BFS properties
      1). # of nodes in a row = queue size before poping up this row's first element.

  2. Examples

    1. Binary Tree Postorder Traversal

       /**
 * Postorder is same as the reverse of right domniant preorder. 
 */
class Solution {
    public List<Integer> postorderTraversal(TreeNode root) {
        List<Integer> ll = new LinkedList<>();
        if(root == null) return ll;
        Stack<TreeNode> st = new Stack<TreeNode>();
        st.push(root);
        while(st.size() != 0){
            TreeNode node = st.pop();
            ll.add(0, node.val);
            if(node.left != null){
                // right is  domniant
                st.push(node.left);
            }
            if(node.right != null){
                st.push(node.right);
            }
        }
        return ll;
    }
}
