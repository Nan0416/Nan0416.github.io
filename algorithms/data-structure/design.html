    1. 146. LRU Cache

      460. LFU Cache

      Desc:
      #146. LRU (least recently used) cache with a certain capacity and will remove the least recently used value once it exceed its capacity.
      #460. LFU (least frequently used) cache with a certain capacity and will remove the least frequently used value once it exceed it capacity. If serveral value are least frequently, then the least recently used value will be removed.

      Sol: #146. HashMap with a self defined LinkedList. HashMap key is key, value is linkedlist node, where a node has {key, value, next, previous} when a node is get/update/insert, we put its node to the front.
      #460. Create a linkedlist node, and a doubly linkedlist (using dummy node) that provides removeNode, removeLastNode, insertIntoFront.
      Two hashmap, a map {} (key = key, value = linkedlist node {key, value, freq, next, previous})
      A frequency map {key = frequency, value = doubly linedlist.}

    4. 295. Find Median from Data Stream

      Desc: given a stream of numbers (in range of [0,255]). Take each number at a time, and return the current median, e.g. [1,2,3], after input 1, median is 1; after input 2, median is 1.5; after input 3, median is 2.

      Sol:
      1). keep a arraylist sorted (find position with binary search, and insert), return median in O(1)
      2). use two priorityqueues, always keep the first values in the two queues are the median candidates.
