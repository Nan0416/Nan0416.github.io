  1. Introduction

    Webpack is a tool for bundling JS module. ES6 defines module and import to make the development of JS application scalable. However, most of broswers do not support this feature yet. So webpack is a tool to "compile" multiple JS module into a single file without "import". Moreover, webpack can minimize the JS files and combine and divide modules into different .js files.

    Functions

    1. minimize js, css
    2. npm install browser js module, and use "import" to import into your code.

    Concepts

    1. Entry

    2. Source Map

      After webpack build the project, code is minimized as a single file. The generated code is almostly not programmer-readable.
      It also increases the difficulty of debugging because we cannot locate the bug in the source code.
      Source Map is a tool for mapping the "compiled" code and the source code. So when a bug happens, it can locate the bug in the source code. 
                          module.exports = {
                        mode: 'development',
                        devtool: 'inline-source-map',
                        ...
                    }

      How does source map work

      The source map is a json data that maps the source code and the compiled js. The JSON data is generated by a tool (e.g. UglifyJS) and can be understood by broswers.
      e.g. At the end of the compiled js file, there is //# sourceMappingURL= that points to the JSON data.

  2. Example

    This example use eventemitter3 library,

    1. Install dependencies

      Install dependencies just like Node.js

                              npm install --save eventemitter3

    2. A project structure

                              /
                            webpack.config.js // webpack configuration file.
                            node_modules/
                            package.json
                            /src
                                /js
                                    core.js
                                /css
                                    .sass
                                    .css
                            /assets
                                .png
                            /dist // the folder for distribution
                                .html
                                .js

    3. Define webpack.config.js

      In order to use webpack, you need to specify a node.js file that defines how to compile your code.

                                  const path = require('path');

                            module.exports = {
                                entry: './src/index.js',
                                output: {
                                    filename: 'main.js',
                                    path: path.resolve(__dirname, 'dist')
                                }
                                mode: 'development',
                                devtool: 'inline-source-map',
                            };
