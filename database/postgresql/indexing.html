  1. Introduction

    Indexes are data structure to facilitate the query process. It provides a secondary access path. A relation can have any number of indexes on arbitrary columns.

    Indexes is like a map data structure, where the key is the indexed column(s), the key is a pointer to disk of the row.

    1. Data file: the file that stores the table.
    2. Primary Index: designate a superkey as the order for a table's physical storage. In other words, the primary index gives an order of the data file.
      Primary index is not necessary to use the fields of the primary key, but it has to be unique.
    3. Clustering index: designate a non unqiue field(s) as the order for the data file.
      Because each table can only have one physical store, then primary index and clustering index cannot use together.
    4. Ordered data file vs. Heap file: A table's physical store can be unordered, in this case, it's called a heap file.
      Insertion & Deletion of an ordered file are slow because it has to shift record. Using a heap file can be quick because it just appends new record at the end of the file.
      Postgres database uses heap file.
    5. Secondary index: an index create on any fields.

    Dense & sparse only for ordered indexes
