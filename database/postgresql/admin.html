  2. Role Vs User

    PostgreSQL database uses Role to manage users. A role is like a Linux user, which has specific privileges. When installing PostgreSQL, a OS user named postgres and a DB role named postgres will be created. When you login Postgre, postgre will use the role has the same name as the current OS user by default. So first switch to OS user postgres, "sudo -i -u postgres", and then login, "psql".

    Postgres Password

    PostgreSQL sets password for neither OS postgres nor DB postgres. The administor needs to setup them
    $sudo passwd postgres
    postgres=> ALTER USER postgres WITH PASSWORD 'new_password';

    Create a role

    CREATE ROLE name LOGIN; # create a role that can login. same as CREATE USER name;

  3. Login Authentication

    PostgreSQL login authentication is controlled by /etc/postgresql/10/main/pg_hba.conf.

    1. Example

                          #type   database-name   role-name   source-address     authentication-method
                    local   all             all                                     peer      
                    host    all             all             127.0.0.1/32            md5
                            # IPv6 local connections:
                            host    all             all             ::1/128                 md5
                            # Allow replication connections from localhost, by a user with the
                            # replication privilege.
                            local   replication     all                                     peer
                            host    replication     all             127.0.0.1/32            md5
                            host    replication     all             ::1/128                 md5
                            host    gps-tracker     postgres        0.0.0.0/0               md5
    2. PostgreSQL distinguishes the behavior of local login (local, unix domain socket) and remote login (host, socket).
      local login does not need a source address
    3. 0.0.0.0/0 matches all
    4. authentication method
      md5: using role password to login
      peer: the OS username same as Role name. #this is why another OS user (nan) cannot login DB role postgres. After change this field to md5, nan can login postgres. ** But only one auth method can be used for a rule **

  4. pgAdmin

    pgAdmin is a GUI for managing PostgreSQL. In server mode, it is run a python2 flask web server on the backend. The "backend" can run everywhere, such as on the same machine of postgresql, same machine of the frontend browser, or an independent machine.

    Download and install through Python Wheel
    Before connecting to a PostgreSQL server, make sure the server pg_hba.conf is configured to be allowed to connect from the backend machine.

    $psql -U
