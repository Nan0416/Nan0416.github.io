  1. Introduction

    DbContext's DbSet implemented IEnumerable, so EF can use LINQ to query data.

    1. Tracking/No-Tracking

      Tracking (default behavior) persisted change made to entity instance after calling .SaveChanges().
      No-tracking: read-only, change will not be persisted.
      Tracking behavior will setup a change tracker, so it is slower then no-tracking.

    2. A query is executed when invoking .ToList, .ToArray, Single, Count.

    3. Include & ThenInclude: Loading Related Data

      By default, the navigation properties is not set. To populate them, using .Include and .ThenInclude

      Eager loading: the Included data is loaded when the query is executed.

                              // one -> one
                        var actors = _context.FilmActor.Where(fa =>fa.FilmId == 141).Include(fa=>fa.Actor);
                        foreach (var actor in actors){
                            Console.WriteLine(actor.Actor.FirstName);
                        }
                        // one -> many
                        var citiesInChina = _context.Country.Where(c =>c.Country1.ToLower() == "china").Include(c => c.City); // .ThenInclude(cities=> ...)
            
                        foreach( var china in citiesInChina){
                            var cities = china.City.ToList();
                            foreach(var city in cities){
                                Console.WriteLine(city.City1);
                            }
                        }

                        // Include/ThenInclude does not support many -> 
                        // e.g. Cannot then include on the City field, because it is a collection.
