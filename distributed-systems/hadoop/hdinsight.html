  1. Introduction

    Microsoft Azure provides a HDInsight cluster service. This service supports Hadoop, Spark and other big data processing tools

    This service is build on top of a cluster of linux machines and a storage.

    Storage

    In order to use this service, we need to create a separate storage service. There are 3 different types of storage services:
    Blob storage (used with HDInsight): Blob storage does not support Hadoop Streaming API
    Storage (used with HDInsight): general storage v1, and general storage v2,
    Queue storage
    ...

    A storage is connected to a cluster. A storage can have multiple container. A container will be initialized as a HDFS.

    Azure Storage
	containers
		HDFS

  2. Usage

    HDInsight storage

    1. Web interface: Azure web portal provide a GUI to create new container, upload files
    2. azure classic cli (run on remote client)
    3. HDInsight hdfs command: Because HDInsight is built on linux, hdfs command can be run on the namenode of the cluster.
      ssh sshuser@nqin8-ssh.azurehdinsight.net
      hdfs dfs -cat wasbs://second-container@nqin8.blob.core.windows.net/movies.csv
      # hdfs dfs -cat wasbs://<container-name>@<account-name>.blob.core.windows.net/movies.csv
