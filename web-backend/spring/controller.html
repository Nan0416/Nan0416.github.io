  2. Mapping & Parameters

    Mapping

    @RequestMapping is used to annotate controller class/method to let them handle centain request.

    1. Url Matching 
                              // url = "/stock/meta/baba" will method the getId method.
                        @RequestMapping(path = "/stock") // path is alias of value.
                        public class Controller{
                            @RequestMapping(value = "/meta/{symbol}") // symbol is a placeholder.
                            public string getId(@PathVariable("symbol") String symbol){
                                return "21343209";
                            }
                        }
    2. Http method: by default, if the request mapping maps all http methods. 
                              @RequestMapping(value = "/stock/meta/{symbol}", method = RequestMethod.GET) // only allow GET
                        @RequestMapping(value = "/stock/meta/{symbol}", method = {RequestMethod.GET, RequestMethod.POST}) // allow GET and POST

                        // ** if both of them appear, it will cause runtime error due to ambiguous.
    3. HTTP URL Parameters 
                              @RequestMapping(value="/meta", params = {"symbol", "!company", "member=premium", "period!=10"})
                        // map url, must have symbol, not have company, member and its value = premium
                        // period can be ignore, but if has period, its value cannot be 10.

                        // meta?symbol&member=premium&period=11
                        // meta?symbol=12&member=premium&period
                        // meta?symbol=12&member=premium

    4. Others

      HTTP request headers, media types in request, accept media types.
