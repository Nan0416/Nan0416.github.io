  1. Introduction

    When the application is runnng, the Main function will create a host instance, which is responsible for app startup and lifetime management. The Startup class specify how to configure middleware and controller/MVC.

    Host

    1. The default used host is Kestrel with enabling IIS integration ???
    2. Kestrel receive http requests and create corresponding HttpContext object to invoke Application's middleware.

  2. Kestrel

    1. Basic

          using Microsoft.AspNetCore;
    using Microsoft.AspNetCore.Hosting;
    public class Application
    {
        public static void Main(string[] args)
        {
            IWebHostBuilder hostBuilder = WebHost.CreateDefaultBuilder(args); // create a kestrel instance.
            hostBuilder.UseStartup<Startup>();
            IWebHost host = hostBuilder.Build();
            host.Run();
        }
    }

      ASP.NET Core use Kestrel as the default web (http) server. Kestrel instance is created when calling .CreateDefaultBuilder(args);

      Kestrel web server helps handle HTTPS, HTTP/2. It is based on the libuv library (Node.js event-loop)

      .CreateDefaultBuilder(args);

      1). .UseKestrel(); // use Kestrel as the web server
      2). .UseIISIntegration(); // enable IIS integration???
      2). .UseContentRoot(Directory.GetCurrentDirectory()); // content root is the directory that contains the application content files, .GetCurrentDirectory() is the current working directory.
      3). load IConfiguration from 'appsettings.json' and 'appsettings.[EnvironmentName].json'; //

    2. Configuration Mechanism

      ASP.NET Core provides a built-in configuration framework that takes a variety of sources, such as .json files, .xml files, environment variables, and command-line arguments.

                              // 
                        IConfigurationBuilder configBuilder = new ...();
                        configBuilder.AddCommandLine(args); // .AddCommandLine is a extension method on the IConfigurationBuilder interface;
                        IConfiguration config = configBuilder.Build(); // interface defined method;

      2). The ".CreateDefaultBuilder(args)" loads configuration from appsettings.json, environment variables, command line arguments, and other sources.
      3). This configuration will be encaspulated in IConfiguration object, which is avaliable to the Startup class.

                                  public Startup(IConfiguration configuration)
                            {
                                Configuration = configuration;
                                Console.WriteLine(configuration["AllowedHosts"]); // come from appsettings.json
                                Console.WriteLine(configuration["name"]); // come from args command-line --name=qinnan 
                            }
