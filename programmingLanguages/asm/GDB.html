  1. Introduction

    GDB is a linux cmd tool to dynamic observe the program in running. (observe the process)

    GDB can help disassembly code, add break point, check registers' values.

  2. Command

    gdb --args ./executable

    1. start: start running the program, it stop at the first default breaking point (main)
    2. disassemble: print the disassembly of current function
    3. disassemble function_name: print the disassembly of the function. e.g. disassembly main
    4. info breakpoints: print all break points
    5. info registers: print all value of registers
    6. break label: break point at label e.g. break main
    7. break *addr: break at the specific addr. e.g. break *0x104c
    8. delete num: delete the breakpoint e.g. delete 2 // delete the second breakpoint
    9. stepi: run one instruction
    10. continue: run the program until next break point
    11. print
