    1. Constructor

      If the developer does not define any constructor, then the compiler provides it a default constructor with no parameters.

      A developer-defined constructor or a compiler-provided constructor must explicitly or implicitly call one of the parent’s constructor at the beginning.
      1).A compiler-provided constructor calls the parent’s constructor that has no parameters. It indicates that the parent must have a constructor without parameter if one of its child has a compiler-provided constructor.
      2). A developer-defined constructor either explicitly invokes one of the parent’s constructors or the compiler let the constructor implicitly invoke the parent’s constructor that has no parameters. So it indicates that if a developer does not explicitly invokes one parent’s constructor, then the parent class must have a constructor without parameters.
