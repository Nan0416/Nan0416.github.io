    String

    String

    1. constructor: (char [])
    2. .charAt(int): char
    3. .toCharArray(): char []
    4. .equals(String): boolean
    5. .substring(int s, int e): String
    6. + (concate): String
    7. .length(): int
    8. .split(String): String[]
    9. .toLowerCase()/ .toUpperCase(): String[]
    10. .indexOf(String): int

    StringBuilder

    1. constructor: (int init_cap), (String str)
    2. .append(String / char / char[]): self
    3. .insert(int offset, char / char[] / String): self. Offset in terms of string
    4. .reverse(): self
    5. .substring(int s, int e):String
    6. .length(): int

    Experiences

    1. Construct/Modify a string: if the size is known, then use char[] and new String(char[]). Otherwise, using StringBuilder.
    2. for loop a string: .charAt()
    3. upper case: (char)(char_c - 'a' + 'A') Return int

    LinkedList

    1. .add(int index, E e), .remove(int i) // out of bounds error
    2. .push(E e); .pop():E // no such element error Push/Pop is on the first (left, index 0) element
    3. .removeFirst(); .add(E e) // queue
    4. .set(index i, E e): replacement
    5. all: .removeFirst/Last() .getFirst/Last() .addFirst/Last(E e)

    TreeMap

    1. for-loop is ordered in an ascending order.
    2. .ceilingEntry(K key): Map.Entry // return a entry's key >= input key May return null
    3. .ceilingKey(K key): K
    4. .floorEntry(K k), .floorKey(K k)
    5. .firstKey, firstEntry Error when map is empty
