  1. Introduction

    Typescript is a superset of Javascript, which means a javascript program is also a typescript program, but a typescript program may not be a javascript program.

    TS is developed by Microsoft.

    Typescript helps develop large project with JS framework.

    Typescript supports classes, modules, and arrow function ECMAScript 2015.

    How to run

    1). using tsc compile .ts file into js file.
    2). using Node.js or browser environment to run the .js file

    tsconfig.json

    tsconfig.json tells tsc how to compile the project.

    {
    "files": [
        "src/main.ts"
    ],
    "compilerOptions": {
        "noImplicitAny": true,
        "target": "es5"
    }
}

    use Gulp to automate compiling, testing

    $npm init
$npm install -g gulp-cli
$npm install --save-dev typescript gulp gulp-typescript # install for development phase

# organize code as following
project/
    src/
    dist/
#write .ts file in src, output .js in dist

    In order to use Gulp, a js code should be supplied to gulp to tell it how to do the automation.
    create gulpfile.js under the project directory 

    // an example
var gulp = require("gulp");
var ts = require("gulp-typescript");
var tsProject = ts.createProject("tsconfig.json");

gulp.task("default", function () {
    return tsProject.src()
        .pipe(tsProject())
        .js.pipe(gulp.dest("dist"));
});

  2. Syntax

    1. Optional type after variable:

          function greeting(value: string):string{
        return "Hello, " + value;
    }
    let value:string = "Nan";
    console.log(greeting(value));

    2. Basic types:

          boolean: false, true
    string: "hello!"
    number: 1, 1.23 
    number[]: [1,2,3]
    [number, string, number]: [1, "one", 1]; // order matter

    3. Special types:

      1).any: Just link pure JS variable, this variable can be any type. Because of the freedom, it can cause runtime error.

      let value:any = 10;
console.log(value);
value = "string";
function printing(input: number){
    console.log(input + 1); 
    // even though the input is expected to be a number, and then perform arithemtical addition.
    // but since "any" can be an input, then the input can be an actual string.
}
printing(value); // string1

//runtime error
let value:any = "string";
console.log(value.toFixed()); // .toFixed is a method for decimal number, but compiler does not check during compiling phase, this caused a runtime error.

    4. Type assertion: a way to tell the compiler "trust me, I know what I’m doing."

      class Data{
    special_function(){
        console.log("Special function");
    }
}
let value:any = "string";
(value as Data).special_function(); // this also work without type assertion.

  3. Q&A

    1. Why needs Type?
      Types help enhance code quality and understandability. When project becomes big, types are important to maintain code.
      It’s better for the compiler to catch errors than to have things fail at runtime.
