  1. Introduction

    .NET IO: provide a set of functions that transfer data between storage medium (disk, memory, network, pipe) and data (a string, a byte[]) in sync and async manner.

    1. File: In C#, file objects used to creating, deleting, get metadata about a file that stored in file systems. It is not designed to read/write data, instead, it can help create a FileStream object to read/write data.
    2. Stream: a stream is used to handle IO in byte[]. The abstract class Stream does not define the underlying physical data source, but it defines common operations, such as read/write/seek/close/flush
      Concrete implemenation includes:

      1).FileStream,
      2).MemeoryStream // Creates a stream whose backing store is memory.
      3).BufferedStream // Adds a buffering layer to read and write operations on another stream.
      4).NetworkStream // Provides the underlying stream of data for network access.
      5).PipeStream // supports both anonymous and named pipes.
    3. Reader/Writer: A stream is a stream of bytes (binary), a reader/writer is converter that decode/encode data. C# provides different kind of reader/writer, including
      1). StreamReader/StreamWriter: reading and writing characters by using an encoding value to convert the characters to and from bytes.
      2). StringReader/StringWriter: reading and writing characters to and from strings. StringReader directly read from a string, StringWriter can write string into an optional StringBuilder.
      3). TextReader/TextWriter: serve as the abstract base classes for other readers and writers that read and write characters and strings, but not binary data. e.g. Json
